    Behavior is specified by Task 2's tests (same TDD cycle: write tests first, then this module). Summary:
    - Level order: phone, then poliza (only if poliza_id given), then global; first block wins and returns its scope.
    - Keys: `rl:phone:{sha256(phone)[:16]}`, `rl:poliza:{sha256(poliza_id)[:16]}`, `rl:global`.
    - Disabled via settings -> immediate `RateLimitResult(True, None)` without touching Redis.
    - Redis exception -> allowed (fail-open) + `log.error("rate_limit.redis_failed", ...)`.
  </behavior>
  <action>
    Create `app/security/rate_limiter.py`:

    1. `_SLIDING_WINDOW_LUA` — RESEARCH Pattern 5 script MODIFIED to return the count for alerting: after ZREMRANGEBYSCORE + ZCARD, `if count >= limit then return -1 end`; else ZADD (member `tostring(now) .. '-' .. tostring(math.random())`), `EXPIRE key math.ceil(window / 1000) + 1`, `return count + 1`. ARGV: now_ms, window_ms, limit. Window passed in milliseconds; expire computed in seconds.
    2. `RateLimitResult` NamedTuple per the interface block.
    3. `T_RATE_LIMITED = "Estás enviando muchos mensajes. Por favor espera un momento e intenta de nuevo."` — lives here (not in qa/messages.py) so this plan owns zero shared files.
    4. `_key(scope: str, raw: str) -> str` — `f"rl:{scope}:{hashlib.sha256(raw.encode()).hexdigest()[:16]}"` (Pitfall 2). Global key is the constant `"rl:global"`.
    5. `async def check_rate_limit(redis, *, phone, poliza_id=None) -> RateLimitResult`:
       - `if not settings.rate_limit.enabled: return RateLimitResult(True, None)`.
       - `now_ms = int(time.time() * 1000)`; `window_ms = settings.rate_limit.window_s * 1000`.
       - Build level list: `[("phone", _key("phone", phone), phone_limit)]` + poliza level if poliza_id + `[("global", "rl:global", global_limit)]`.
       - For each level: `result = await redis.eval(_SLIDING_WINDOW_LUA, 1, key, now_ms, window_ms, limit)` inside try/except Exception -> on exception `log.error("rate_limit.redis_failed", scope=scope, error_type=...)` and CONTINUE to next level (fail-open). `int(result) == -1` -> `log.warning("rate_limit.exceeded", scope=scope, limit=limit)` and return `RateLimitResult(False, scope)`. Else if `int(result) >= ceil(limit * 0.8)` -> `log.warning("rate_limit.approaching", scope=scope, count=int(result), limit=limit)` (Sentry picks up via structlog/sentry integration — no direct sentry_sdk call needed).
       - All levels passed -> `RateLimitResult(True, None)`.
    6. `__all__ = ["check_rate_limit", "RateLimitResult", "T_RATE_LIMITED"]`.

    Never log raw phone/poliza — only scope + counts. mypy --strict clean (redis param typed Any with comment, same convention as meta.py `_dispatch_message`).
  </action>

  </verify>
