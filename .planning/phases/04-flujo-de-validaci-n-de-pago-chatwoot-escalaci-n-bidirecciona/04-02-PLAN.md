    - validate_magic_bytes(b"RIFF\x00\x00\x00\x00WEBPVP8", "image/webp") == True
    - validate_magic_bytes(b"%PDF-1.4", "application/pdf") == True
    - validate_magic_bytes(b"MZ\x90\x00", "image/jpeg") == False (.exe spoofed as jpeg)
    - validate_magic_bytes(b"%PDF-1.4", "image/jpeg") == False (mime mismatch)
    - validate_magic_bytes(b"\xff\xd8\xff", "video/mp4") == False (disallowed declared mime — D-24)
    - ATTACHMENT_MAX_BYTES == 5 * 1024 * 1024
    - ALLOWED_MIME_TYPES == frozenset({"image/jpeg","image/png","image/webp","application/pdf"})
  </behavior>
  <action>
    Create `app/features/payment/attachment.py`:

    - `ATTACHMENT_MAX_BYTES: Final[int] = 5 * 1024 * 1024  # 5 MB per D-25`
    - `ALLOWED_MIME_TYPES: Final[frozenset[str]] = frozenset({"image/jpeg", "image/png", "image/webp", "application/pdf"})  # D-24`
    - `_MAGIC: Final[dict[str, tuple[bytes, ...]]] = {"image/jpeg": (b"\xff\xd8\xff",), "image/png": (b"\x89PNG\r\n\x1a\n",), "image/webp": (b"RIFF",), "application/pdf": (b"%PDF-",)}`

      - If `declared_mime not in ALLOWED_MIME_TYPES`: return False.
      - Look up signatures: `sigs = _MAGIC[declared_mime]`.
      - Return True iff `any(first_chunk.startswith(sig) for sig in sigs)`.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| app → Meta Graph API | Outbound HTTPS bearing WA_TOKEN; existing TLS verification kept |
| Meta CDN → app | Inbound binary content; magic-byte validated before storage |
| client → app | Comprobante uploaded by client lands here; mime can be spoofed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-02-01 | Spoofing | Mime-spoof attack (.exe declared as image/jpeg) | mitigate (D-26) | `validate_magic_bytes` blocks before disk write; callers in Wave 3 MUST gate on this. |
| T-04-02-02 | Tampering | Meta CDN URL substitution | accept | TLS pinning out of scope; Meta domain verified by bearer token + standard CA chain. |
| T-04-02-03 | Information disclosure | WA_TOKEN logged | mitigate | Token never logged; only `media_id`, `mime`, `size`, `_hash_phone(to)` go to structlog. |
| T-04-02-04 | DoS | Oversize attachment download | mitigate (D-25) | `download_media` checks `file_size` from metadata BEFORE the binary GET; rejects >5 MB. |
| T-04-02-05 | Elevation of privilege | Comprobante content reaches LLM | mitigate (D-27) | `download_media` returns raw bytes only to payment node; never tools-exposed to LLM. Enforced by call-site discipline in Plan 04-04. |
| T-04-02-SC | Tampering | New package install | mitigate | No new packages added; validator is stdlib-only manual byte check (per ponytail ladder rung 5). |
