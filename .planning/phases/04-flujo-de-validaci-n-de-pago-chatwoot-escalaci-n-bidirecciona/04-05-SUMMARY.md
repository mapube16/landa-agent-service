# Phase 04 Plan 05: Cartera Routing Branch + Payment Graph Resume Helper Summary

Cartera-allowlist branch wired into Meta webhook _dispatch_message, and Command(resume=) helper that resumes the LangGraph payment interrupt from cartera button taps.

## Objective

Close the loop between Plan 04-04 (graph suspends at `node_awaiting_cartera` via `interrupt()`) and the actual cartera taps that should resume it. Without this plan, cartera button taps land in the regular client Q&A handler (or get bounced by the client allowlist).

## What Was Built

### Task 1: cartera.py helper module

`app/features/payment/cartera.py`:

**`parse_button_id(raw: str) -> tuple[str, str] | None`**
- Parses `"<action>|<case_id>"` button ids generated by `node_forward_to_cartera`
- `VALID_ACTIONS = frozenset({"aprobar","rechazar","info"})` gates all action strings
- Returns `None` on malformed input (no pipe, unknown action, empty case_id)
- Security T-04-05-05: free text never reaches this function; only parsed on interactive taps

**`resume_payment_interrupt(*, action, case_id, extra, qa_graph, db_session_factory) -> bool`**
- Loads `Case` from DB by `case_id`; returns `False` if not found
- Idempotent on terminal statuses (`approved/rejected/escalated/closed`) — returns `True` without touching graph (T-04-05-03)
- Resumes via `Command(resume={"action":action,"extra":extra})` — the LangGraph 1.2.x recommended pattern for `interrupt()` resumption; `node_awaiting_cartera` receives this as the return value of `interrupt()`
- `config = {"configurable": {"thread_id": case.phone}}` — thread_id is client's phone (checkpoint key)

**`handle_cartera_message(*, msg, qa_graph, meta_client, db_session_factory) -> None`**
- Interactive button tap: `parse_button_id` → if valid, call `resume_payment_interrupt`; else fall through to re-send
- Text / unrecognized button / fallthrough: find latest `awaiting_cartera` case, re-send 3 action buttons
- No open case: silent log + drop (T-04-05-04)

15 tests in `test_cartera_handler.py` — all green.

### Task 2: Meta webhook cartera-allowlist routing branch

`app/webhooks/meta.py` changes:

**`_get_cartera_allowlist() -> frozenset[str]`** (new module-level function)
- `@lru_cache(maxsize=1)` — reads `settings.payment.cartera_phone_allowlist` once at startup
- Tests patch this function directly (avoids lru_cache staleness between test runs)
- Returns `frozenset` of E.164 strings from `CARTERA_PHONE_ALLOWLIST` env var

**`_dispatch_message` cartera branch** (inserted before client allowlist):
```
dedup → cartera-allowlist → client-allowlist → firewall → graph dispatch
```
- `normalized_from = _normalize_e164(msg.from_)` — reuses existing helper
- `if normalized_from in cartera_allow:` → `await handle_cartera_message(...)` + `return`
- Unknown numbers (not cartera, not client) → silently dropped at existing `is_echo_allowed` check

Updated invariant docstring in module header, `receive()` endpoint docstring, and `_dispatch_message` docstring.

4 tests in `test_meta_cartera_routing.py` — all green.
