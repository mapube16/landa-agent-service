        - log info `scheduler.reminder_sent` with `case_id`, `elapsed=elapsed_minutes`.
        - Continue (do not escalate in the same tick).

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-04-06-01 | Tampering | Cron fires twice within one minute (Railway restart) | mitigate | `reminder_sent_at IS NULL` predicate prevents double-reminder; `escalated_at IS NULL` predicate plus `status='escalated'` write makes escalate idempotent. |
| T-04-06-02 | Information disclosure | Cleanup unlinks attachments before disputes resolved | accept (D-02) | 90-day retention is the policy; ajustable when DPG legal redefines. Documented in code comment + CONTEXT D-02. |
| T-04-06-03 | DoS | Polling query without index causes table scan | mitigate | Plan 04-01 added partial index `ix_cases_work_hours_due_at WHERE status='awaiting_cartera'`. |
| T-04-06-04 | Elevation of privilege | Scheduler escalates the wrong case | mitigate | Predicate keyed on case_id; UPDATE filtered by case_id; single-row WHERE per branch. |
| T-04-06-05 | Repudiation | Reminder sent but operator claims it wasn't | mitigate | `reminder_sent_at` timestamp + `scheduler.reminder_sent` structlog event with case_id; F5 audit log will append-only mirror. |
