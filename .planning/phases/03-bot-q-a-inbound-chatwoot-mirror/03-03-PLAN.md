## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Webhook handler (03-05) → ARQ enqueue_job | kwargs cross via JSON serialization; only primitives allowed (Pitfall 6) |
| ARQ worker → Chatwoot Application API | `api_access_token` header in every request; SecretStr keeps it out of logs |
| ChatwootClient → Redis cache | conv_id cached 7d under phone_hash; cache failure bypasses cleanly (read errors logged + skip) |
| Chatwoot API response → ChatwootClient | Untrusted JSON; only `contact.id` and `conv.id` extracted — no eval, no exec, no full passthrough to logs |
