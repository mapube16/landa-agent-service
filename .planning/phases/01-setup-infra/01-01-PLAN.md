<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| dev → git remote | Code + lockfile cross to GitHub; secrets must not |
| CI runner → PyPI | Untrusted package downloads during `uv sync` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-01-01 | Information disclosure | .env / secrets committed | mitigate | `.gitignore` excludes `.env`, `.env.*`, `*.pem`, `*.key`; pre-commit `detect-private-key` hook blocks key material at commit time |
| T-01-02 | Tampering | Slopsquatted dependency | mitigate | All 28 packages already verified via slopcheck in RESEARCH.md Package Legitimacy Audit (all OK, three HALLUCINATION_PATTERN flags suppressed for langchain-openai/sentry-sdk/python-multipart with rationale documented). pyproject pins exact versions; `uv.lock` adds hash verification |
| T-01-03 | Tampering | Unpinned transitive deps | mitigate | `uv sync --frozen` rejects any deviation from `uv.lock`; CI uses `--frozen` |
| T-01-04 | Information disclosure | Large file accidentally committed (e.g. dump of secrets) | mitigate | pre-commit `check-added-large-files --maxkb=500` |
| T-01-SC | Tampering | npm/pip/cargo installs | accept | F1 only installs from PyPI with pinned versions + hash-verified lockfile. No `[ASSUMED]`/`[SUS]` packages per RESEARCH.md audit. slopcheck CI gate is deferred to F5 per CONTEXT.md scope. |
