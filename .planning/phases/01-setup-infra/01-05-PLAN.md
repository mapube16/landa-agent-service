## Trust Boundaries

| Boundary | Description |
|---|---|
| Internet → agent-app public domain | TLS terminated at Railway edge; all traffic public until F2 adds HMAC |
| agent-app ↔ agent-postgres | Internal Wireguard mesh (`railway.internal`); no public egress for DB |
| agent-app ↔ agent-redis | Same as Postgres |
| Operator local CLI → Railway control plane | OAuth/PAT; never commit `railway` config tokens |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-01-22 | Information disclosure | Postgres or Redis exposed publicly | mitigate | Railway provisions databases on internal network by default; never enable public networking until F5 audit clears it |
| T-01-23 | Information disclosure | Credentials committed in railway/*.json | mitigate | JSON snapshots use placeholders; CI grep gate (future) catches `sk-or-`/`ls-`/`@.*ingest.sentry` patterns |
| T-01-24 | Information disclosure | /health public, can leak component error type | accept | Returns only `error: <ExceptionTypeName>` (already mitigated in plan 04); message bodies suppressed |
| T-01-25 | Denial of service | Unauthenticated /test/llm burns OpenRouter quota from public domain | accept (F1 scope) | F5 hardens; for F1 the endpoint is needed for E2E verification |
| T-01-26 | Spoofing | Webhook URLs not yet active (Meta integration is F2) | N/A | No webhooks exposed in F1 |

</threat_model>
