    **Archivo 1: `app/config/logging.py`** — structlog routed via stdlib + key-name+regex redaction + correlation_id (per CONTEXT.md D-07 + RESEARCH.md Pattern 3 + Pitfall 7).

    Implementar:

    ```python
    import logging, re, sys
    from typing import Any
    import structlog
    from structlog.contextvars import merge_contextvars

    PII_KEYS: frozenset[str] = frozenset({
        "phone", "phone_number", "wa_token", "wa_phone_id", "wa_webhook_secret",
        "openrouter_api_key", "langsmith_api_key", "sentry_dsn",
        "softseguros_password", "softseguros_username",
        "lambda_proyect_internal_token", "chatwoot_api_key",
        "saldo", "saldo_pendiente", "documento", "cedula", "monto",
        "credit_card", "cvv", "ssn",
    })

    PHONE_RE = re.compile(r"\+?\d[\d\s\-]{7,}\d")

    def redact_pii(_, __, event_dict: dict[str, Any]) -> dict[str, Any]:
        for k in list(event_dict):
            if k.lower() in PII_KEYS:
                event_dict[k] = "[REDACTED]"
        for k, v in event_dict.items():
            if isinstance(v, str):
                event_dict[k] = PHONE_RE.sub("[REDACTED_PHONE]", v)
            elif isinstance(v, dict):
                event_dict[k] = redact_pii(None, None, v)
        return event_dict

    def configure_logging(log_level: str = "INFO", env: str = "dev") -> None:
        timestamper = structlog.processors.TimeStamper(fmt="iso", utc=True)
        shared = [
            merge_contextvars,
            structlog.processors.add_log_level,
            timestamper,
            redact_pii,
            structlog.processors.StackInfoRenderer(),
            structlog.processors.dict_tracebacks,
        ]
        # Pretty for dev tty, JSON for non-tty (Railway logs)
        renderer = (
            structlog.dev.ConsoleRenderer()
            if env == "dev" and sys.stdout.isatty()
            else structlog.processors.JSONRenderer()
        )
        structlog.configure(
            processors=shared + [
                structlog.stdlib.ProcessorFormatter.wrap_for_formatter,
            ],
            logger_factory=structlog.stdlib.LoggerFactory(),
            wrapper_class=structlog.make_filtering_bound_logger(
                getattr(logging, log_level.upper(), logging.INFO)
            ),
            cache_logger_on_first_use=True,
        )
        # Route stdlib logging (uvicorn, sqlalchemy, etc) through same processor chain
        formatter = structlog.stdlib.ProcessorFormatter(
            foreign_pre_chain=shared,
            processors=[
                structlog.stdlib.ProcessorFormatter.remove_processors_meta,
                renderer,
            ],
        )
        handler = logging.StreamHandler(sys.stdout)
        handler.setFormatter(formatter)
        root = logging.getLogger()
        root.handlers.clear()
        root.addHandler(handler)
        root.setLevel(log_level.upper())
        # Silence noisy loggers
        for noisy in ("uvicorn.access", "httpx", "httpcore"):
            logging.getLogger(noisy).setLevel(logging.WARNING)
    ```

    Notas:
    - `redact_pii` MUST come before any renderer/formatter (RESEARCH.md Pattern 3 inline comment).
    - Recursive walk para nested dicts es necesario porque tool outputs y request bodies vienen anidados.
    - `merge_contextvars` permite que asgi-correlation-id inyecte `correlation_id` en el contextvar y aparezca en cada log line — el wiring de la middleware vive en `main.py` (plan 04).

    **Archivo 2: `app/config/observability.py`** — Sentry init con auto-detection FastAPI + scrubber + langsmith env presence check (per CONTEXT.md D-07 + RESEARCH.md Pattern 4):

    ```python
    from typing import Any
    import sentry_sdk
    from sentry_sdk.integrations.starlette import StarletteIntegration
    from sentry_sdk.integrations.fastapi import FastApiIntegration
    from sentry_sdk.integrations.asyncpg import AsyncPGIntegration
    from sentry_sdk.integrations.redis import RedisIntegration
    from app.config.settings import settings
    from app.config.logging import PII_KEYS, PHONE_RE

    def _scrub_value(v: Any) -> Any:
        if isinstance(v, str):
            return PHONE_RE.sub("[REDACTED_PHONE]", v)
        if isinstance(v, dict):
            return {k: ("[REDACTED]" if k.lower() in PII_KEYS else _scrub_value(val)) for k, val in v.items()}
        if isinstance(v, list):
            return [_scrub_value(x) for x in v]
        return v

    def scrub_sentry_event(event: dict[str, Any], hint: dict[str, Any]) -> dict[str, Any] | None:
        # Scrub headers, query strings, body, exception frames, breadcrumbs
        for path in ("request", "contexts", "breadcrumbs", "exception"):
            if path in event:
                event[path] = _scrub_value(event[path])
        # Aggressive: drop any frame local var with PII-shaped key
        for ex in event.get("exception", {}).get("values", []):
            for frame in ex.get("stacktrace", {}).get("frames", []):
                if "vars" in frame:
                    frame["vars"] = {
                        k: ("[REDACTED]" if k.lower() in PII_KEYS else _scrub_value(v))
                        for k, v in frame["vars"].items()
                    }
        return event

    def init_sentry() -> None:
        if not settings.sentry.dsn:
            return  # Sentry deshabilitado (tests, CI sin DSN)
        sentry_sdk.init(
            dsn=settings.sentry.dsn.get_secret_value(),
            environment=settings.app.app_env,
            release=settings.app.app_version,
            traces_sample_rate=settings.sentry.traces_sample_rate,
            profiles_sample_rate=settings.sentry.profiles_sample_rate,
            send_default_pii=False,
            integrations=[
                StarletteIntegration(transaction_style="endpoint"),
                FastApiIntegration(transaction_style="endpoint"),
                AsyncPGIntegration(),
                RedisIntegration(),
            ],
            before_send=scrub_sentry_event,
        )
    ```

    Notas críticas:
    - `init_sentry()` debe llamarse ANTES de instanciar FastAPI app y antes de importar los routers (RESEARCH.md Pattern 4 anti-pattern). El wiring de la llamada vive en `app/main.py` (plan 04) como primer statement después de los imports.
    - NO usar `sentry_sdk.integrations.asgi.SentryAsgiMiddleware` manualmente — auto-detection en sentry-sdk 2.x lo hace y la wrap manual rompe `request.body()` (RESEARCH.md Pattern 4 gotcha).
    - `send_default_pii=False` es no-negociable (CONTEXT.md security domain).

<threat_model>

## Trust Boundaries

| Boundary | Description |
|---|---|
| process → stdout (Railway log sink) | Logs cross trust boundary; PII must be scrubbed in-process |
| process → Sentry SaaS | Errors exfiltrate to SaaS; PII must be scrubbed before send |
| .env → settings → in-memory | Secrets in memory but never serialized (SecretStr renders as `**********`) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-01-05 | Information disclosure | structlog leaks PII to stdout | mitigate | `redact_pii` processor with key-name + regex hybrid (per D-07); runs before renderer; recursive walk for nested dicts |
| T-01-06 | Information disclosure | Sentry exfiltrates request bodies / exception locals with PII | mitigate | `send_default_pii=False` + `before_send=scrub_sentry_event` that walks request/contexts/breadcrumbs/exception frames |
| T-01-07 | Information disclosure | uvicorn.access logs include client phones in URLs | mitigate | Set uvicorn.access logger level WARNING; structured access logs (with redaction) come later F5 |
| T-01-08 | Tampering | Manual SentryAsgiMiddleware double-wraps and breaks request.body() | mitigate | Use auto-detection (no manual middleware); explicit anti-pattern called out in code comments |
| T-01-09 | Information disclosure | SecretStr fields serialized via Pydantic .dict() leak values | mitigate | Never dump full Settings to logs; SecretStr renders as `**********` by default — code review |
| T-01-SC | Tampering | Container base image (python:3.12-slim) | accept | Pulled from Docker Hub official; F5 may add image scanning (trivy) in CI |
